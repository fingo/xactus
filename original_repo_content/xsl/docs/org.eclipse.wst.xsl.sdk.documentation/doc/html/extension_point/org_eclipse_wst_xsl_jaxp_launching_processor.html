XSLT Processor

Identifier:
org.eclipse.wst.xsl.launching.jaxp.processor

Description:
Contribute a Java XSLT processor instance (e.g. Xalan 2.7.0). The extension point primarily defines the classpath to be used when launching an XSLT transformation.

Configuration Markup:

<!ELEMENT extension (processor+)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED

>


<!ELEMENT processor EMPTY>

<!ATTLIST processor

id              CDATA #REQUIRED

label           CDATA #REQUIRED

processorTypeId CDATA #REQUIRED

classpath       CDATA #REQUIRED

debuggerId      CDATA #IMPLIED

supports        CDATA #REQUIRED

>

An instance of a JAXP-compliant XSLT processor


  • id - A unique id for this processor instance
  • label - A label for this instance to be displayed in the UI
  • processorTypeId - The id for the processor type
  • classpath - The classpath to use when this processor is launched
  • debuggerId - The id of a debugger to associate with the processor
  • supports - A comma-separated list of supported XSLT versions e.g. 1.0,2.0

Examples: 

   <extension
         point="org.eclipse.wst.xsl.jaxp.launching.processor">
      <!-- An adopter can specify the ${eclipse_orbit: } variable to indicate which eclipse
           bundle should be added to the class path. -->
      <processor
      classpath="${eclipse_orbit:org.apache.xalan};${eclipse_orbit:org.apache.xml.serializer};${eclipse_orbit:org.apache.bcel};${eclipse_orbit:java_cup.runtime}"
            id="org.eclipse.wst.xsl.launching.xalan.processor"
            label="%processor.label.1"
            processorTypeId="org.eclipse.wst.xsl.xalan.processorType"
            debuggerId="org.eclipse.wst.xsl.xalan.debugger"
            supports="1.0">
      </processor>
   </extension>
